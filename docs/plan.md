# 特价机票发现平台 — 完整需求方案

> 版本：v2.0（深度复盘后全面更新）
> 日期：2026-04-12
> 定位：国内用户的「出行情报站」，帮助用户高效发现和选择特价机票

---

## 一、产品定位与核心价值

### 1.1 我们是什么

**出行情报站，不是比价工具。**

携程、去哪儿已经把比价做得很好了。我们不是要替代它们，而是做它们没做好的事：

> 把分散在各处的「出行情报」聚合起来，帮用户在下手之前做出更聪明的决策。

用户的真实流程是：
```
发现机会 → 判断值不值 → 去平台购买
```

我们负责前两步，购买交给携程/去哪儿。

### 1.2 用户真实痛点（来自小红书调研）

1. **信息分散**：要省钱必须同时盯着携程、去哪儿、飞猪、各航司官网，没有人帮你聚合
2. **时机难把握**：不知道现在买还是等等，不知道哪天最便宜
3. **促销日记不住**：东航18号、春秋27号、南航28号……各航司会员日分散，用户靠脑子记
4. **大数据杀熟**：同一航班刷新一次涨200，用户已形成反制意识但操作繁琐
5. **目的地选择难**：有预算有时间，但不知道能飞哪里

### 1.3 我们能解决什么

| 用户痛点 | 我们的解法 | 能解决程度 |
|---|---|---|
| 信息分散 | 聚合热门航线最低价 Feed | ✅ 部分解决（数据来自飞常准，非全平台） |
| 时机难把握 | 未来价格日历，看哪天最便宜 | ✅ 有效解决 |
| 促销日记不住 | 航司会员日日历 + 倒计时 | ✅ 完全解决 |
| 大数据杀熟 | 透明展示参考价，引导多平台比价 | ✅ 部分缓解 |
| 目的地选择难 | 按预算反向推荐目的地 | ✅ 有效解决 |

### 1.4 我们不能解决什么（诚实说明）

- **实时精准价格**：飞常准数据有延迟，不是携程/去哪儿的实时价。我们展示的是「参考价」，用户需要去平台确认最终价格。这一点必须在产品中明确告知，不能误导用户。
- **直接购票**：我们不是 OTA，不做购票，只做「发现」和「决策辅助」。

## 二、用户画像

### 2.1 核心用户群

**价格敏感型旅行者（主力用户）**
- 年龄：19-35岁，一二线城市
- 特征：出行前1个月内才决定，对价格高度敏感，有薅羊毛意识
- 痛点：不知道现在买还是等，担心被杀熟
- 使用场景：打开我们网站，快速判断「这条航线近期有没有低价机会」

**计划型旅行者**
- 特征：提前规划，日期灵活，愿意为最低价调整出行时间
- 痛点：要在多个平台反复查不同日期，太麻烦
- 使用场景：用价格日历一眼看出哪天最便宜，然后去携程下单

**灵感型旅行者**
- 特征：有出行意愿但目的地不固定，以预算为出发点
- 痛点：不知道同样的钱能飞哪里
- 使用场景：输入预算，发现「原来¥500能飞三亚」

### 2.2 用户比价行为（来自调研）

用户口诀：
> 天巡/谷歌航班比价 → 携程/飞猪二次议价 → 航司官网查会员价 → 叠加平台优惠券

我们的产品定位在这个流程的**最前端**——帮用户发现值得关注的机会，然后引导他去对的平台完成购买。

## 四、数据层设计

### 4.1 数据来源现实

经过充分调研，国内机票数据对个人开发者的可用性如下：

| 数据源 | 可用性 | 说明 |
|---|---|---|
| 携程 API | ❌ 不可用 | 无公开 API，B2B 商业合作才能接入 |
| 去哪儿 API | ❌ 不可用 | 无公开 API |
| 飞猪 API | ❌ 不可用 | 需阿里生态认证合作商 |
| 中航信 TravelSky | ❌ 不可用 | 只对持牌旅行社开放 |
| GitHub 爬虫 | ❌ 全部失效 | 2017-2019年项目，反爬机制已升级 |
| **飞常准 MCP API** | ✅ 可用 | 中国本土民航数据公司，个人可注册 |

**结论：飞常准是目前唯一合法、可用、覆盖国内航线的个人开发者数据源。**

### 4.2 飞常准数据说明

- **数据来源**：中航信（TravelSky）授权 + 航司直连 + 部分 OTA 抓取
- **数据性质**：参考价，非实时价，有一定延迟
- **价格准确性**：方向性准确（哪天便宜/哪天贵），精确价格需去平台确认
- **费用**：每次调用 ¥0.5（50点），新用户赠送5000点（¥50），充值享充1送4优惠

### 4.3 价格透明度策略

**核心原则：诚实告知，建立信任，而不是隐瞒误导。**

在以下位置明确标注：
1. 首页顶部横幅：「价格来自飞常准，为参考价，实际以购票平台为准」
2. 每张航班卡片：价格旁标注「参考价」小字
3. 价格日历页：日历下方说明文字
4. 目的地灵感页：结果列表上方说明

**这样做的好处**：用户知道这是参考价，跳转到携程发现价格有差异时，不会觉得被骗，反而会理解这是正常的价格波动。

### 4.4 API 调用优化

飞常准按次计费，需要节省调用：

- **服务端缓存**：相同查询结果缓存1小时，同一航线当天只调用1次
- **Mock 数据兜底**：额度耗尽时自动切换预置真实数据，Demo 不白屏
- **按需加载**：价格日历默认加载14天，用户翻页再加载更多
- **热门航线预设**：首页 Feed 只查询预设12条热门航线

### 5.1 跳转目标平台

**不跳转 Google Flights**（国内用户不用），改为：

| 按钮 | 跳转目标 | URL 格式 |

## 六、UI 设计要求

### 6.1 整体风格

- 简洁、现代、有设计感，不是功能堆砌
- 主色调：蓝色系（信任感）+ 橙红色（价格标签、促销感）
- 字体：中文优先，清晰易读
- 卡片式布局，信息层次清晰

### 6.2 关键视觉元素

- **价格**：最大最醒目，红色，用户第一眼看到
- **价格标签**：🔥超低价（红）/ ✅低价（绿）/ 正常（灰）
- **参考价标注**：小字灰色，不抢眼但清晰可见
- **会员日标记**：🎯图标，橙色，在日历和侧边栏突出显示
- **跳转按钮**：携程用蓝色，去哪儿用橙色，视觉区分

### 6.3 加载体验

- 骨架屏（Skeleton）替代 loading spinner
- 价格日历先展示框架，价格数据异步填入
- Mock 数据模式下顶部显示黄色提示条

## 八、已确认决策

| 事项 | 决策 | 原因 |
|---|---|---|
| 数据源 | 飞常准 MCP API | 唯一合法可用的个人开发者国内航班数据源 |
| 价格性质 | 参考价，明确标注 | 无法获取实时价，诚实告知建立信任 |
| 购票跳转 | 携程 + 去哪儿（双按钮） | 国内用户实际使用的平台，Google Flights 不适用 |
| 航线范围 | 仅国内航线 | 飞常准国内数据最准，国际线另有更好数据源 |
| 用户登录 | 不需要 | 零门槛，降低使用门槛 |
| 部署 | Vercel | 一键部署，免费额度够用 |
| 界面语言 | 中文 | 面向国内用户 |
| 设备优先级 | 桌面优先 | 复杂信息展示在桌面体验更好 |
