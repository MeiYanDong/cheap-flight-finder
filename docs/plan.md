# 特价机票发现平台 — 完整需求方案

> 版本：v1.1（已根据调研结论更新）
> 日期：2026-04-10
> 定位：桌面优先、无需登录、接入真实 API 的国内机票 Demo 产品

---

## 一、产品背景与目标

### 1.1 背景

各大 OTA 平台（携程、去哪儿、飞猪）和航司频繁推出特价机票活动，但存在以下核心问题：

- **信息分散**：用户需要在十几个平台间手动比价
- **大数据杀熟**：平台根据用户设备、历史记录动态调价（2026年4月监管新规已明确禁止）
- **时机把握难**：各航司会员促销日规律不同，用户需要手动记忆和追踪
- **行动滞后**：发现特价时往往已经售罄，错过时间窗口
- **判断困难**：不知道当前价格在未来几天是涨是跌，不敢下手

### 1.2 产品目标

打造一个**透明、高效、无需登录**的国内特价机票发现平台，帮助用户：

1. 第一时间发现当前最低价国内航班
2. 通过未来价格日历找到最便宜的出行日期
3. 掌握航司促销规律，提前布局
4. 按预算反向发现目的地

### 1.3 核心差异化

| 差异点 | 说明 |
|---|---|
| 未来价格日历 | 展示未来60天每天最低价热力图，替代无法获取的历史数据 |
| 航司会员日日历 | 聚合各航司促销日，可视化展示倒计时 |
| 透明全价展示 | 含税费真实全价，不隐藏附加费 |
| 无需登录 | 零门槛使用 |

### 1.4 数据来源说明（重要）

经过充分调研，国内机票数据现实情况如下：

- 携程、去哪儿、飞猪**均无公开开发者 API**，数据完全封闭
- **历史价格数据**：国内无任何面向个人开发者的历史价格 API，此功能已从产品中移除
- **选定数据源**：VariFlight（飞常准）MCP API
  - 中国本土航班数据公司，国内航线是核心业务
  - 提供真实可购买航班和最低价数据
  - 注册地址：mcp.variflight.com，免费100次调用，个人可注册
- **"去购买"跳转**：跳转 Google Flights 对应搜索结果页（中立、无偏向、URL 可携带参数）

## 二、用户画像

### 主要用户群体

**价格敏感型旅行者**
- 年龄：19-35岁，一二线城市为主
- 特征：出行前1个月内才决定订票，对价格高度敏感
- 痛点：不知道现在买还是等等，担心被大数据杀熟
- 需求：快速发现低价，知道未来哪天更便宜

**计划型旅行者**
- 特征：提前规划，希望找到特定航线的最低价日期
- 痛点：需要在多个平台反复查询不同日期
- 需求：价格日历，一眼看出哪天最便宜

**灵感型旅行者**
- 特征：有出行意愿但目的地不固定，以预算为出发点
- 痛点：不知道同样的钱能飞哪里
- 需求：按预算反向推荐目的地

### 3.2 P0 — 首页：特价 Feed

首页以"发现"为核心，展示当前全网最低价国内航班列表，用户无需输入任何信息即可获得价值。

#### 页面布局

```

├─────────────────────────────────────────────────────┤
│  Hero区：今日特价标语 + 快速筛选栏                   │

├──────────────────────┬──────────────────────────────┤
```

#### 特价航班卡片设计

每张卡片包含：
- 出发地 → 目的地（城市名 + 机场代码）
- 出发时间 + 到达时间 + 飞行时长
- 航司名称 + 航班号
- **当前价格**（含税全价，醒目大字）
- **折扣率**（如：3折、5折，来自 VariFlight 返回字段）
- 价格标签：🔥 超低价 / ✓ 低价 / 准点率（来自 VariFlight 准点率字段）
- "查看日历"按钮 → 跳转价格日历页
- "去购买"按钮 → 跳转 Google Flights 对应搜索页

#### 筛选功能

| 筛选项 | 选项 |

| 出发城市 | 北京 / 上海 / 广州 / 深圳 / 成都 / 杭州 / 重庆（可搜索） |

- 显示"数据更新时间"
- 提供手动刷新按钮
- API 响应缓存5分钟（节省 VariFlight 调用次数）

### 3.3 P0 — 价格日历：未来60天最低价热力图

#### 功能描述

用户选定出发地和目的地后，以日历热力图形式展示未来60天每天的最低价格，帮助用户快速找到最便宜的出行日期。这是替代"历史价格曲线"的核心差异化功能——用户不需要知道过去多少钱，他们需要知道**未来哪天最便宜**。

#### 页面布局

```
┌─────────────────────────────────────────────────────┐
│  搜索栏：[出发地] ⇄ [目的地] [单程/往返] [搜索]     │
├─────────────────────────────────────────────────────┤
│  价格日历（热力图）                                  │
│                                                     │
│  四月                    五月                        │
│  一  二  三  四  五  六  日  ...                     │
│  ¥-  ¥-  680 720 ¥-  890 ¥-  ...                   │
│  （颜色深浅表示价格高低，绿=便宜，红=贵）             │
│                                                     │
├─────────────────────────────────────────────────────┤
│  选中日期的航班列表                                  │
│  [航班卡片1] [航班卡片2] [航班卡片3]...              │
└─────────────────────────────────────────────────────┘
```

#### 热力图规则

- 颜色梯度：深绿（最低价）→ 浅绿 → 黄 → 橙 → 红（最高价）
- 无航班日期：灰色显示
- 鼠标悬停：显示当日最低价 tooltip
- 点击日期：下方展示该日所有可用航班列表
- 最低价日期加"最低"角标

#### 往返模式

- 支持切换单程 / 往返
- 往返模式下展示两个日历（去程 + 返程）
- 自动高亮"总价最低组合"

#### 灵活日期对比

- 提供"±3天"弹性对比表格
- 帮助用户发现"提前一天出发能省多少钱"

## 四、技术架构

### 4.1 技术栈

| 层级 | 技术选型 | 理由 |
|---|---|---|
| 前端框架 | Next.js 14 (App Router) | SSR/SSG 支持，SEO 友好，Vercel 一键部署 |
| UI 组件库 | shadcn/ui + Tailwind CSS | 高质量组件，定制灵活 |
| 图表库 | Recharts | React 生态，价格折线图/热力图 |
| 状态管理 | Zustand | 轻量，适合无登录场景 |
| 数据请求 | TanStack Query (React Query) | 缓存、自动刷新、loading 状态 |
| 后端 API | Next.js API Routes | 同一项目，无需独立后端，隐藏 API Key |
| 真实数据 | VariFlight MCP API | 中国本土，覆盖国内航线，有最低价数据 |
| 部署 | Vercel | 一键部署，环境变量管理 |

### 4.2 VariFlight MCP API 使用规划

| 功能 | API 调用方式 | 对应页面 |
|---|---|---|
| 特价 Feed | 查询热门航线最低价 | 首页 |
| 价格日历 | 循环查询未来60天每天价格 | 价格日历页 |
| 航班详情 | 查询特定航班信息 | 详情页 |
| 目的地灵感 | 查询出发城市到多个目的地的最低价 | 灵感页 |
| 城市搜索 | 城市/机场代码查询 | 搜索框自动补全 |

### 4.3 API 调用优化策略

由于免费额度有限（100次），采用以下策略：

- **缓存**：相同查询结果缓存5分钟（Next.js Route Handler + 内存缓存）
- **预设热门航线**：首页 Feed 只查询预设的20条热门航线，不做全量查询
- **懒加载**：价格日历按月加载，不一次性查询60天
- **去重**：相同出发地+目的地+日期的查询合并

## 五、页面交互设计细节

### 5.1 搜索框交互

- 支持城市名（中文）和机场代码（IATA）输入
- 实时自动补全（基于本地城市列表，无需额外 API 调用）
- 出发地/目的地一键互换按钮
- 记住用户最近搜索的城市（localStorage）

### 5.2 加载状态设计

- 骨架屏（Skeleton）替代 loading spinner，减少视觉跳动
- 价格日历：先展示日历框架，价格数据异步填入
- 错误状态：友好提示 + 重试按钮

### 5.3 价格展示规范

- **所有价格均为含税全价**，页面明确标注"含税费"
- 价格单位：人民币（¥），保留整数
- 折扣率：保留一位小数（如 4.5折）
- 数据来源：标注"数据来源：飞常准"，更新时间

点击"去购买"跳转 Google Flights 搜索结果页，URL 格式：
```
https://www.google.com/travel/flights?q=从{出发城市}到{目的地城市}{日期}的航班
```
理由：中立平台，不偏向任何 OTA，用户信任度高。

### 5.5 响应式设计

- 桌面优先（1280px 基准宽度）
- 平板适配（768px）：侧边栏收起，主内容全宽
- 移动端基础可用（不作为重点优化目标）
